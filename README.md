# Family Task Manager

A mobile-first, real-time task management application designed for families to organize chores, track progress, and manage household responsibilities efficiently.

---

## 🏗 Architecture Overview
The project follows a **Monorepo architecture** managed via `pnpm workspaces`, separating the concerns of the frontend and backend while maintaining unified linting and configuration.

*   **Frontend:** A responsive React application optimized for mobile usage.
*   **Backend:** A robust FastAPI service handling business logic and data persistence.
*   **Database:** MongoDB for flexible schema management, integrated with Supabase for authentication and real-time synchronization.
*   **Communication:** RESTful API for standard CRUD operations, augmented by Supabase Realtime for instant updates across client devices.

---

## 🛠 Technology Stack
*   **Frontend:** React, Vite, Tailwind CSS, TanStack Query, Framer Motion.
*   **Backend:** FastAPI, Pydantic, Motor (Async MongoDB driver).
*   **Database:** MongoDB (via MongoDB Atlas).
*   **Auth & Realtime:** Supabase.
*   **Deployment:** Vercel (Frontend & Serverless Functions).

## 🏁 Getting Started

### Prerequisites
*   [Node.js](https://nodejs.org/) (v18+)
*   [Python](https://www.python.org/) (v3.10+)
*   [pnpm](https://pnpm.io/)
*   [Docker](https://www.docker.com/) (for local DB development)

### Local Setup
1. **Clone the repository:**
   ```bash
   git clone <repo-url>
   cd family-task-manager
   ```

2. **Install dependencies:**
   ```bash
   pnpm install
   ```

3. **Configure Environment:**
   Copy the `.env.example` to `.env` in both `/frontend` and `/backend` directories and populate your Supabase and MongoDB credentials.

4. **Start Development:**
   ```bash
   # Start the database and backend
   docker-compose up
   # Start the frontend
   pnpm --filter frontend dev
   ```

## 📝 Implementation Notes

*   **Real-time Updates:** We utilize Supabase Realtime to listen for database changes. React Query is used to manage the cache; when a change is detected via the subscription, we trigger a `queryClient.invalidateQueries` to ensure the UI stays in sync.
*   **RBAC:** The FastAPI backend uses dependency injection (`Depends`) to verify user roles (Parent vs. Child) before allowing task modifications or deletions.
*   **Optimistic Updates:** Frontend interactions (like toggling a task as complete) use React Query's optimistic updates to provide instant feedback to the user before the server response confirms the action.
*   **Security:** All API routes are protected by JWT validation. The frontend automatically attaches the Supabase-provided JWT to the `Authorization` header via Axios interceptors.
